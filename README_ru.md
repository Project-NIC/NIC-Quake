## Что такое QUAKE?

QUAKE — это референсный проект открытого оборудования для распределённой сейсмографической сети. Каждый узел представляет собой герметичный автономный модуль, предназначенный для постоянной установки в скале, фундаменте здания и т.д. — залитый полиуретановым компаундом, подключённый водонепроницаемым разъёмом M12, и оставленный работать на годы.

Проект возник из простого разочарования: существующие подходы либо стоят целое состояние (геофоны), требуют точной механической установки, либо компенсируют шумные дешёвые датчики горами программного обеспечения. QUAKE идёт противоположным путём — потрать больше на аппаратуру, получи чистый сигнал сразу, программная часть остаётся простой.

## Почему не геофон?

- Стоит столько же, сколько весь узел QUAKE
- Плохая низкочастотная характеристика — крутой спад усиления ниже резонанса, необходимы сложные корректирующие усилители с фазовыми ошибками
- Три отдельных устройства для трёх осей
- Требует точного выравнивания и механической установки

## Почему не ёмкостные датчики?

- Пробная масса должна быть тяжёлой для чувствительности, но тяжёлая масса означает низкую собственную резонансную частоту и лёгкое возбуждение
- Паразитные ёмкости к стенкам корпуса меняются с температурой, влажностью и геометрией
- Результат: нестабильный сигнал, отражающий условия внутри корпуса, а не сейсмическую активность

## Почему MEMS?

Пробная масса MEMS микроскопична — собственный резонанс находится в диапазоне кГц, сейсмика — ниже 20 Гц. Они никогда не мешают друг другу. Датчик герметично запаян на одном кремниевом кристалле, откалиброван на заводе, цифровой выход, три оси в одном корпусе. Один чип заменяет три геофона за долю стоимости.

## Как это работает

### Виртуальное выравнивание — устанавливай как угодно

Никаких уровней. Никаких регулировочных винтов. Никакой точной механической установки.

Бросаешь модуль в отверстие в скале, отправляешь команду калибровки, процессор записывает текущее состояние датчиков как референсный ноль. Вектор гравитации автоматически определяет абсолютную ориентацию — ось Z перпендикулярна ядру Земли независимо от физического положения. Матрица преобразования делает остальное.

### Аппаратная синхронизация часов

Все узлы используют единый физический тактовый сигнал, распределяемый мастером по второй паре RS-485. Это не программная метка времени и не NTP — это реальный тактовый сигнал на проводе. Неопределённость синхронизации — наносекунды, не миллисекунды.

### Топология сети

Стандартный кабель UTP Cat 6 (уличное исполнение), четыре пары — две для данных и тактового сигнала RS-485, две для питания. Топология daisy-chain, каждый узел имеет два разъёма M12 IP68 (вход и выход), согласующие резисторы только на каждом конце линии. До 8 узлов на сегмент.

### Модульный дизайн

Каждый датчик использует один интерфейс SPI от MCU с собственным выводом Chip Select. Универсальная печатная плата предусматривает все три позиции датчиков — ненужный датчик просто не устанавливается. Прошивка автоматически определяет установленные датчики при запуске через SPI. Перекомпиляция не требуется.

## Стек данных

| Уровень | Библиотека | Роль |
|---|---|---|
| Сжатие | [NIC-DMD](https://github.com/Project-NIC/NIC-DMD) | Адаптивное сжатие, без таблиц, работает на ATmega328 |
| Хранение | [NIC-MLA](https://github.com/Project-NIC/NIC-MLA) | Однофайловый контейнер, crash-safe, self-describing |
| Шифрование транспорта | [NIC-KSF](https://github.com/Project-NIC/NIC-KSF) | SPECK-128 CTR, in-place, без malloc |
| Запись | [NIC-GLUE-IN](https://github.com/Project-NIC/NIC-GLUE-IN) | Данные датчиков → DMD → MLA |
| Чтение/экспорт | [NIC-GLUE-OUT](https://github.com/Project-NIC/NIC-GLUE-OUT) | MLA → CSV / SQLite |
| Просмотр | [NIC-VDE](https://github.com/Project-NIC/NIC-VDE) | MLA как каталог, экспорт в CSV/SQL |

## Статус проекта

**Прошивка: переносимое ядро готово и протестировано (v0.5.0)** — драйверы датчиков (включая температуру каждого датчика и изменение диапазона в реальном времени), автоопределение, калибровка осей, протокол RS-485 и конечный автомат узла, всё протестировано вне целевого чипа; слой STM32 (board) написан и ждёт плату. **Аппаратура:** референсный дизайн определён, схемы в работе. Следующие шаги — PCB и полевые испытания.

Вклад приветствуется — аппаратура, прошивка или программное обеспечение.
